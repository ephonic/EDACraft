## 2. 修复清单（按优先级与依赖排序）

### P0 — 影响正确性（必修）

#### P0-1 Transient 内层 Newton 三件套
**文件**：`src/solver/time_stepper.cpp`
**改动**：
1. 把 `for (lit < 10)` 改为参数化 `opts.localNewtonMaxIter`（默认 50）。
2. Newton 步前加 `dvmax` 限幅（仅对节点电压列）。
3. 加 Armijo 回溯线搜索（接受准则 `f1 ≤ f0·(1 - 1e-4·α)`）。
4. `localConv == false` 时不再 `(void)localConv`，而是返回错误：
   ```cpp
   if (!localConv) {
       r.diags.error({}, "transient: inner Newton failed at t=" + ...);
       return r;
   }
   ```
5. 在 `TimeStepperOptions` 增加 `dvmax`、`localNewtonMaxIter`、`gmin`（已有）等字段。

**验证**：68 测试 + 强非线性 BSIM4 transient 用例。

#### P0-2 Transient 入口 `resetLimiting`
**文件**：`src/solver/time_stepper.cpp`
**改动**：在 `integrateTransient` 入口（初始化设备状态前）加：
```cpp
for (const auto& d : devices)
    if (auto* o = dynamic_cast<OsdiModel*>(d.get())) o->resetLimiting();
```
**风险**：极低 —— 已在 DC/HB 入口验证有效。

#### P0-3 Shooting 入口 + FD 扰动前 `resetLimiting`
**文件**：`src/solver/shooting.cpp`
**改动**：
1. 入口同 P0-2。
2. 每次 FD 扰动 `restoreDeviceStates(devices, savedStates)` **后**调一次 `resetLimiting`，让两次扰动从相同 limiting 锚点出发。

**风险**：低；可能略微改变 FD 列数值（更准）。

#### P0-4 Shooting 内层 Newton 严格 Armijo
**文件**：`src/solver/shooting.cpp:165`
**改动**：把
```cpp
if (f1 <= f0 * (1.0 + 1e-10) || lambda < 1e-4) accept;
```
改为：
```cpp
if (f1 <= f0 * (1.0 - 1e-4 * lambda)) { accept; break; }
if (lambda < 1e-4) { /* 用 best 步或退出 */ }
```
保留"取最佳已试步长"作为 fallback。

### P1 — 影响鲁棒性

#### P1-5 Shooting 内层 gmin log-spaced homotopy
**文件**：`src/solver/shooting.cpp`
**改动**：
1. 在 `ShootingOptions` 增加 `gminStart`、`gmin`、`gminSteps`。
2. 内层 Newton 包一层 homotopy 循环（与 DC 相同模式）。
3. 失败时回退到上一已收敛 gmin。

**注意**：会让单步时间增加；要观察 BSIM4 PSS 是否仍在 60s 内完成。

#### P1-6 DcOpResult 未收敛时调用侧兜底
**文件**：`src/solver/time_stepper.cpp`、`src/solver/shooting.cpp`、`src/solver/hb_nonlinear.cpp`
**改动**：检查 `dc.converged`，若 false：
- 打印 warning；
- 用 `dc.nodeVoltages`（best-converged）作为初值，而非零；
- 不要静默继续。

#### P1-7 Transient/Shooting stagnant/floor 检测
**文件**：`time_stepper.cpp` / `shooting.cpp`
**改动**：把 DC 的 stagnant 检测复用过来：
```cpp
const bool stagnant = (alpha < 1e-3) && (fDelta < fOld*1e-3) && fOld > 0.0;
if (stagnant) accept_as_step_solution;
```

#### P1-8 统一 gmin 配置语义
**文件**：所有 solver hpp
**改动**：抽出 `struct GminOptions { gmin, gminStart, gminSteps };` 复用。
**风险**：API 变动；测试需要同步。

### P2 — 影响精度 / 可观测性

- **P2-11** ✅ DC 提取支路电流改为 KCL-LSQ：保留 `nodeV+dx` 节点电压 polish（BSIM4 → HB-NL warm start 必需），但支路电流改用 `M·I_VS = -F*` 在 polish 后的 V* 上做最小二乘 KCL 闭合，不再依赖 augmented Newton 的 `dx[br]` 读数。简化版"直接砍 +dx"会破坏 (V, I) 自洽性导致 BSIM4 HB 不收敛，已实测验证。68/68 通过。

## 5. 风险与回滚

每项改动单独 commit。出现回归：
1. 先看是否仅仅是某测试 tolerance 太紧 → 调宽；
2. 若是真发散 → revert 该 commit，记录原因，进 P2/P3 阶段重新设计。

特别注意：
- **P0-1** 改 transient 内层是高风险区，必须分小步走，每个子改动都跑测试。
- **P1-5** Shooting gmin homotopy 会让单步时间倍增，先实现 `gminSteps=0` 的零开销路径作为兜底。
