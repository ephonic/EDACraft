### 0.2 现有收敛性测试形态

- 单元测试粒度：每个测试只断言 `r.converged == true` 或 `r.nodeVoltages` 有限。
- 偏置/温度/参数扫描下的**收敛率统计**：无。
- 参数 corner（W/L/Vdd/Temp）网格：无。
- 多器件级联（diff pair / current mirror / cascode / OTA）：无。
- 射频典型电路（LC oscillator / mixer / PA stage / LNA）：无。

### 0.3 现有规模与性能可观测性

- 测试电路最大节点数 ≤ ~5（共源、整流器、RC）。
- 无 benchmark 框架；无 N 阶级联 / ladder / multi-finger MOS array。
- GMRES 子空间重用收益、KLU 重因子化时间、内存峰值：均**未度量**。
- 现有 `RFSIM_DCOP_VERBOSE` 是唯一可观测开关。

## 2. 主线分解

### 主线 A — OSDI 模型矩阵填表（直接对应 G1）

#### A1 — bsimsoi DC OP 收敛测试（最缺）
**网表**：4 端 (D, G, S, B)，共源结构，VDD=1.0V，VG=0.6V。
**断言**：`solveDcOp` 收敛；`Id` 在 1nA–10mA 量级；`r.nodeVoltages` 有限。
**风险**：bsimsoi 实例参数表与 bsim4 不同，需要 `bsimsoiModelParams()` helper。

#### A2 — bsimsoi / ekv / bsim4soi / bsimcmg 共源 HB-NL 收敛
**模板**：与 `Bsim4CommonSourceConverges` 同构，`VG=0.55–0.7V`、`VDS=1V`、`AC mag = 0.1 × VG`、`f0=1MHz`、`NH=1`。
**断言**：`r.converged == true`；`|V(d)_H1| < VDD`；输出有限。
**风险**：每个模型在 cutoff/triode 区域对 dvmax / gmin 起跳值敏感；可能需要每模型单独 tune `opts.gmin.gminSteps`、`dvmax`。

#### A3 — diode（完整 Verilog-A）DC + HB
**模板**：与 `simple_diode` 测试同构，但用 `models/diode.dll`。
**断言**：DC OP 收敛、HB 整流谐波非零、与 simple_diode 结果在 5–15% 误差内（容许真实模型有寄生电容差异）。
**风险**：完整 diode 模型可能引入电容/温度依赖，导致 HB warm start 起跳更敏感。

#### A4 — nmos_sh（Verilog-A 短沟道近似）DC + 弱信号 HB
**断言**：DC OP 收敛；HB 在 NH=1 小幅度下收敛。
**风险**：sh 模型可能没有完整的 BSIM 限幅器；需要观察是否触发 dvmax 频繁。

#### A5 — 表格化跑分输出
为每模型单独输出一行：`<model> | dcop=PASS | hb=PASS | shoot=PASS | iters=… | wall=…`，便于 v3 对比。

### 主线 B — 收敛性压测（对应 G2）

#### B1 — 参数扫描收敛率统计 harness
**新增**：`tests/test_convergence_grid.cpp`
- 给定一个参数集合（VG ∈ [0.3, 0.9] step 0.05；VDD ∈ [0.6, 1.5] step 0.1；T = 250/300/400 K）。
- 对每个 (VG, VDD, T) 跑 `solveDcOp` + `solveHbNonlinear`，记录 converged？iters？wall。
- 输出 CSV 到 `build/conv_grid_<model>.csv`。
- 断言：收敛率 ≥ 95%（bsim4 弱非线性区）/ 80%（深亚阈值或深三极管）。

#### B2 — 多器件电路收敛性
**网表**（每条单独 .sp + 对应 GoogleTest）：
- `cascode_amp.sp`：cascode 共源放大器（4 个 MOSFET，6 节点）
- `current_mirror.sp`：5-bit current mirror（10 个 MOSFET）
- `diff_pair.sp`：差分对 + 尾电流源
- `nmos_inverter_chain.sp`：5 级反相器链（CMOS 等效用 BSIM4 + PMOS 模型；若无 PMOS 则用 BSIM4 N + 大电阻 pull-up）

**断言**：每电路 DC OP 收敛 + Shooting-PSS 收敛 + 输出谐波有限。
**风险**：current mirror 的镜像精度对 KCL 余量敏感（P2-11 改动后已变好，但需复核）。

#### B3 — 退化情形与边界
- **悬空节点**：故意构造一个浮空节点（仅 OSDI 内部节点暴露），验证 P3-14 类问题。
- **零电阻 / 极大电阻**（1pΩ, 1TΩ）：验证 gmin floor 行为。
- **负 VDD 或反偏**：验证 limiting 是否反向 latch。
- 断言：要么收敛，要么明确警告 + 不 NaN（不允许静默错误）。

#### B4 — gmin homotopy 路径覆盖
**目的**：之前 plan0620 P1-7 引入的 stagnant 检测路径，需要至少一个测试**真正触发**它（覆盖证明而非纯回归）。
**做法**：构造一个深亚阈值 BSIM4 偏置（VG=0.2V），故意 `gminSteps=20` 让它进入 floor，断言 `r.converged == true` 且 `dcopVerbose=1` 时打印含 "residual floor"。

### 主线 C — 中等规模 + 性能基线（对应 G3 + G4）

#### C1 — 中等规模 RF 典型拓扑
| 拓扑 | 规模 | 模型 | 期望 |
|---|---|---|---|
| **20 段 RC ladder lowpass** | 21 节点 | 内置 R/C | AC + HB 与解析解吻合 |
| **5 级 cascode chain** | ~30 节点 | bsim4 | DC OP + Shooting-PSS 收敛 |
| **1 GHz LC tank + 共源管** | ~6 节点（含 inductor 高 g） | bsim4 + L/C | Shooting 谐波幅度合理 |
| **8-finger MOS multi-instance**（同一模型卡） | ~10 节点，8 实例 | bsim4 | DC OP 收敛；8 个 Id 一致 |

#### C2 — 大规模（节点 ~500）压测
**网表生成器**：写 `tools/gen_ladder.py`（或 .bat + cli），自动产出 N 段 RC ladder（N=100/200/500），N 段反相器链（CMOS）。
**断言**：DC OP wall < 5s @ N=500；Shooting wall < 60s @ N=200。
**目的**：暴露 KLU pattern build / GMRES 子空间内存增长 / shooting FD-Jac 重组装时间。

#### C3 — 性能基线落库
**新增**：`build/bench_<timestamp>.json`，每用例一条：
```json
{ "case": "Bsim4CommonSourcePssConverges",
  "wall_ms": 30240, "newton_iter": 18, "gmres_restarts": 3,
  "klu_factor_ms": 12.3, "klu_solve_ms": 0.4,
  "peak_rss_mb": 87 }
```
**实现路径**：
- `src/solver/dc_op.cpp` / `hb_nonlinear.cpp` / `shooting.cpp` 已有 verbose；新增 `RFSIM_BENCH_JSON=1` 开关把关键计数器以 JSON 写入 stderr。
- GoogleTest fixture 在 `TearDown` 写入 `build/bench_*.json`。
- CI 比对：与上次基线偏离 > 30% 时打 warn（不 fail）。

#### C4 — 性能瓶颈识别
基于 C3 数据，对前 3 个 wall-time 主导项做一次 perf 采样（Windows `xperf` 或 perf-events，见 tools/）。结论写入 `status0620_v3.md`。

## 4. 测试通过判据（汇总）

| 类别 | 通过门槛 |
|---|---|
| OSDI Descriptor 层 | 加载成功 + num_terminals ≥ 4（MOS）/ 2（diode） |
| OSDI DC OP 单点 | `r.converged == true` + `\|V\| < 1e6` + 有限 |
| OSDI HB 单点 | `r.converged == true` + 输出有限 |
| OSDI Shooting 单点 | `r.converged == true` + 末态周期残差 < tol |
| 参数扫描 | 收敛率 ≥ 阈值（弱 95% / 强 80%） |
| 中等规模 (N=20–50) | wall < 30s；Newton iter < 100 |
| 大规模 (N=200–500) | wall 按 C2 表；不允许 NaN/Inf |
| 边界场景 | 收敛 OR 明确 warn；不允许静默 NaN |

## 5. 风险与回退策略

| 风险 | 概率 | 缓解 |
|---|---|---|
| ekv/bsim4soi/bsimcmg/bsimsoi 实例参数 schema 不兼容现有 helper | 中 | 每模型单独 `<model>ModelParams()`；必要时跳过该模型 + 显式 `GTEST_SKIP` 记录原因 |
| diode/nmos_sh 实测 DC OP 不收敛 | 低 | 调整 `gmin.gminSteps`；若仍不行作为已知问题登记，不阻塞主线 |
| 大规模 N=500 在 KLU 上超时 | 中 | 切到 GMRES 直接路径；记入 C4 perf 报告 |
| 参数扫描收敛率不达 95% | 中 | 先记基线，作为 v3 收敛性新一轮入口（不在本计划修复） |
| 性能 JSON 落库引入 IO 开销 | 低 | 仅 `RFSIM_BENCH_JSON=1` 时启用 |
