# Design Compiler Modular Architecture

## Overview

The DC adapter has been refactored into a **configuration-driven, modular architecture** where each feature section is generated by a dedicated function, controlled by YAML config flags.

## Architecture

```
configs/FullSystem.yaml
         ↓
   config_loader.py
         ↓
   DesignState (config)
         ↓
   dc_adapter.py (modular generators)
         ↓
   DC/run.tcl (synthesis script)
```

## Key Components

### 1. Per-Design YAML Config (`configs/*.yaml`)

Each design has its own YAML file specifying:
- **Basic info**: design name, top module, clock period
- **RTL sources**: directory, filelist.f, or explicit files
- **Libraries**: std cell, IO, macro, multi-VT
- **EDA paths**: Synopsys, Mentor environment scripts
- **Synthesis options**: all industrial features as config flags

Contains all industrial-grade DC options:

### 3. Modular Generator Functions

Each function generates one TCL section, controlled by config flags:

```python
class DCAdapter:
    def generate_script(self):
        sections = []
        sections.append(self._gen_header())
        sections.append(self._gen_host_options())
        sections.append(self._gen_variables())
        sections.append(self._gen_svf())
        sections.append(self._gen_libraries())
        sections.append(self._gen_timing_settings())
        sections.append(self._gen_read_rtl())
        sections.append(self._gen_sdc_constraints())
        sections.append(self._gen_timing_derate())
        sections.append(self._gen_physical_constraints())  # DC-T
        sections.append(self._gen_mcmm())                  # MCMM
        sections.append(self._gen_power_optimization())
        sections.append(self._gen_clock_gating())
        sections.append(self._gen_hierarchy_management())
        sections.append(self._gen_vt_control())
        sections.append(self._gen_congestion())
        sections.append(self._gen_dft_scan())
        sections.append(self._gen_design_rules())
        sections.append(self._gen_compile_command())
        sections.append(self._gen_post_compile())
        sections.append(self._gen_reports())
        sections.append(self._gen_write_outputs())
        return "\n".join(sections)
```

Each generator checks config flags and only emits TCL if enabled:

```python
def _gen_physical_constraints(self):
    syn = self.state.config.synthesis
    if not syn.topographical_mode:
        return []  # Skip if disabled
    
    lines = ["# ---- Physical Constraints (DC-T) ----"]
    if syn.tlu_plus_file:
        lines.append(f'set_tlu_plus_files -max_tluplus "{syn.tlu_plus_file}"')
    # ... more DC-T commands
    return lines
```

## Usage

### 1. Create Design Config

```yaml
# configs/MyDesign.yaml
design_name: "MyDesign"
top_module: "top"
clock_period_ns: 10.0

rtl:
  dir: "syn/rtl"

synthesis:
  compile_ultra: true
  num_cores: 8
  
  physical:
    enabled: false
  
  power:
    enabled: true
    leakage: true
    dynamic: true
  
  clock_gating:
    enabled: true
    style: "integrated"
```

### 2. Generate and Run

```bash
# Generate TCL from YAML
python3 -c "
from src.db.config_loader import load_design_config
from src.tools.dc_adapter import DCAdapter

state = load_design_config('configs/MyDesign.yaml')
adapter = DCAdapter(state)
adapter.setup_work_dir()
script = adapter.generate_script()

with open('work/synthesis/DC/run.tcl', 'w') as f:
    f.write(script)
"

# Run DC
source /share/apps/EDAs/syn22.bash
dc_shell -no_gui -f work/synthesis/DC/run.tcl
```

### 3. Enable Industrial Features

Edit YAML to enable features:

```yaml
synthesis:
  # Enable DC-T physical awareness
  physical:
    enabled: true
    tlu_plus_file: "tlu/tt.tluplus"
  
  # Enable MCMM
  mcmm:
    enabled: true
    scenarios:
      - name: "func_tt"
        corner: "tt"
        sdc_file: "sdc/func_tt.sdc"
  
  # Enable scan insertion
  dft:
    scan_insertion: true
  
  # Preserve hierarchy
  hierarchy:
    keep: ["u_CPU", "u_MEM"]
    size_only_patterns: ["pipe_gen_mux"]
```

## Benefits

1. **Configuration-Driven**: All options in YAML, no hardcoded paths
2. **Modular**: Each feature is a separate generator function
3. **Selective**: Only enabled features appear in TCL script
4. **Maintainable**: Easy to add/modify features
5. **Reusable**: Same framework for multiple designs

## Feature Coverage

| Feature | Config Flag | Generator Function |
|---------|-------------|-------------------|
| DC-T Physical | `physical.enabled` | `_gen_physical_constraints()` |
| MCMM | `mcmm.enabled` | `_gen_mcmm()` |
| Power Optimization | `power.enabled` | `_gen_power_optimization()` |
| Clock Gating | `clock_gating.enabled` | `_gen_clock_gating()` |
| Hierarchy Control | `hierarchy.keep` | `_gen_hierarchy_management()` |
| VT Control | `vt_control.dont_use` | `_gen_vt_control()` |
| Congestion | `congestion_optimization` | `_gen_congestion()` |
| DFT/Scan | `dft.scan_insertion` | `_gen_dft_scan()` |
| SVF/ECO | `svf_enabled` | `_gen_svf()` |

## Comparison: Before vs After

### Before (Old Architecture)
- Hardcoded paths: `/share/apps/EDAs/syn22.bash`
- Monolithic script generator
- All features always included
- No YAML config
- Difficult to customize per design

### After (New Architecture)
- Configurable paths: `eda.synopsys`
- 20+ modular generator functions
- Features enabled/disabled via YAML
- Per-design YAML config files
- Easy to customize and reuse
