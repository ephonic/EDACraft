# MoMCraft 使用说明（中文教程）

MoMCraft 是一个基于 **矩量法（Method of Moments, MoM）** 的电磁求解器，
用于提取三维互连结构的 **S 参数**：微带线、多层基板、硅通孔（TSV）、
微凸点，以及 **UCIe（Universal Chiplet Interconnect Express）** 等先进封装。

本指南从环境准备讲起，用三个递进示例带你跑通完整的"几何 → 扫频 → S 参数 →
Touchstone"流程。

## 1. 它解决什么问题

给定一段互连（微带线、过孔、Bump 阵列等）和介质参数，MoMCraft 计算：

- **端口 S 参数**（反射 S11、传输 S21 …），单位为复数；
- **0–60 GHz 宽频带**扫频结果；
- 输出符合 **Touchstone 2.0** 规范的 `.sNp` 文件，可直接导入 ADS / HFSS /
  CST / scikit-rf 等工具。

物理方法：

- **多层介质并矢格林函数**：谱域 TM/TE 反射 + S 矩阵递推；
- **QWE（quadrature-with-extraction）Sommerfeld 积分**：J1 零点 + Shanks 外推；
- **两级 Aksun 尾部提取** + **表面波极点提取**（Chew 搜索 + Hankel 空域）；
- **RWG 基函数**（三角网格）+ **Schur N 端口降阶** + Z→S 变换。

## 2. 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| Python | ≥ 3.9 | 推荐 3.10 / 3.11 |
| NumPy | ≥ 1.24 | 运行时唯一硬依赖 |
| CMake | ≥ 3.16 | 构建系统 |
| C++ 编译器 | C++17 | Windows: MSVC 2019+/2022；Linux/macOS: GCC 9+ 或 Clang 10+ |
| Ninja | 任意 | 推荐的构建生成器（VS 自带；Linux 用 `apt install ninja-build`） |
| Eigen | 3.4.0 | **首次构建自动拉取**（需联网），无需手动安装 |
| OpenMP | 可选 | 启用后矩阵装配 / 扫频并行，加速明显 |

可选依赖：

```bash
pip install gmsh          # UCIe / 3D 网格生成
pip install matplotlib    # 画图
pip install scikit-rf     # S 参数后处理 / 与其他工具互导
```

## 3. 安装与编译

MoMCraft 需要编译 C++ 扩展（`_mom`）。**源码安装**（editable）是首选方式：

### 跨平台（推荐）

在仓库根目录（含 `pyproject.toml`）下执行：

```bash
pip install -e .
```

`scikit-build-core` 会自动调用 CMake / Ninja 编译 C++ 核心并装好 Python 包。

> 首次配置时 CMake 会用 `FetchContent` 从网络拉取 Eigen 3.4.0，请确保联网。

### Windows

```bat
scripts\build.bat
```

该脚本自动探测 `py` 启动器或 `python`，调用 `pip install -e .`，并跑一次冒烟测试。
干净重建用 `scripts\rebuild.bat`（清空 `build/` 与旧 `.pyd` 后重新编译并跑 pytest）。

如果你手动配置（不走 pip），等价命令大致是：

```bat
cmake -S . -B build -G Ninja ^
  -DCMAKE_BUILD_TYPE=Release ^
  -DMOM_BUILD_PYTHON=ON ^
  -DMOM_BUILD_TESTS=OFF
cmake --build build -j
:: 然后把 py\ 加到 PYTHONPATH
set PYTHONPATH=%CD%\py
```

### Linux / macOS

```bash
pip install -e .
```

若要用 GCC/MinGW 而非默认工具链，可设环境变量（见 `CMakeLists.txt`）：

```bash
MOM_USE_MINGW=ON pip install -e .
```

### 验证

```bash
python -c "import mom; print(mom.__version__)"
pytest tests/test_smoke.py -q
```

## 4. 第一个示例：扫频配置 + 冒烟测试

`examples/freq_sweep_demo.py` 演示最基本的 Python↔C++↔NumPy 通路与扫频配置：

```bash
python examples/freq_sweep_demo.py
```

核心代码：

```python
import numpy as np
import mom

print(mom.__version__)                  # 版本
a = np.arange(1, 6, dtype=np.float64)
mom.square_in_place(a)                  # 零拷贝：原地平方
print(a)                                # [1, 4, 9, 16, 25]

lin = mom.FreqSweep.linear(1e6, 10e9, 11)        # 线性扫频
log = mom.FreqSweep.logarithmic(1e6, 10e9, 5)    # 对数扫频
print(lin.frequencies() / 1e9)                   # 返回所有频点（Hz）
```

## 5. 第二个示例：微带线 → Touchstone

`examples/run_touchstone_demo.py` 是最完整的端到端示例：配置几何 → 扫频 →
S 参数 → 写 `.s2p` → 读回验证。

```python
import numpy as np
import mom

ms = mom.Microstrip(
    length=20e-3,      # 走线长 20 mm
    width=3e-3,        # 线宽 3 mm
    height=1.6e-3,     # 介质厚度
    eps_eff=1.0,       # 有效介电常数（1.0 = 自由空间验证）
    nx=40,             # 沿走线方向分段数（越大越精确、越慢）
    gauss=4,           # 每段高斯积分阶数
    z0_ref=50.0,       # 端口参考阻抗
)

freqs = np.linspace(0.5e9, 10e9, 20)
S = ms.solve_sweep(freqs)             # → (nfreq, 2, 2) complex128
ms.to_touchstone("demo_microstrip.s2p", freqs, fmt="RI")

# 读回
fr, S, z0 = mom.read_touchstone("demo_microstrip.s2p")
```

`Microstrip` 主要方法：

| 方法 | 返回 | 说明 |
|------|------|------|
| `solve(freq)` | `(2,2)` complex | 单频点 S 参数 |
| `solve_sweep(freqs)` | `(nfreq,2,2)` complex | 逐点扫频 |
| `to_touchstone(fn, freqs, ...)` | `Path` | 扫频并写 `.s2p` |

## 6. 第三个示例：任意导体（RWG 网格）—— Structure

当结构不是简单矩形微带线时，用 `mom.Structure`：它基于三角网格 + RWG 基函数 +
并矢格林函数 + Schur N 端口，支持任意平面/三维导体（含过孔、垂直电流）。

```python
import numpy as np
import mom

s = mom.Structure(
    conductor=mom.RectangleConductor(
        x0=0, x1=0.02,      # 20 mm 长
        y0=-0.0015, y1=0.0015,  # 3 mm 宽
        z=0.0016,
    ),
    medium=mom.Stackup(eps_r=4.3, h=1.6e-3),
    nx=8, ny=2,
    z0_ref=50.0,
)
print("RWG 基函数数:", s.nb)

s.add_port("in", 0)     # 第 0 个 RWG 基函数作为端口 1
s.add_port("out", -1)   # -1 = 最后一个基函数作为端口 2

S = s.solve(1e9)        # 1 GHz 单频 → (2,2) S 参数
s.to_touchstone("rwg.s2p", np.linspace(1e9, 10e9, 10))
```

> **端口如何选**：`add_port(name, basis_index)` 在某个 RWG 基函数（边）上定义
> 端口。`-1` 表示最后一个。复杂结构（多根耦合线、过孔）可定义 3、4 … N 个端口，
> `solve`/`to_touchstone` 会自动输出对应 `.s3p` / `.s4p` …。

## 7. 高级：32-bit UCIe 互连

UCIe 是芯粒（chiplet）互连标准。MoMCraft 内置 32-bit UCIe 几何构建器
（依赖 `pip install gmsh`）：

```python
import mom.gmsh_mesh

mesh, structure = mom.gmsh_mesh.build_ucie_32bit_mesh(n_bits=32)
print(f"{structure.n_bits}-bit UCIe, {len(structure.port_bumps)} signal bumps")
print(f"网格: {mesh.n_triangles()} triangles")
```

完整流程示例见 `examples/run_ucie_demo.py`。

也可以用 `mom.gmsh_mesh.GmshMesher` 手动拼接任意结构：

```python
from mom.gmsh_mesh import create_gmsh_mesher, finalize_gmsh_mesher

m = create_gmsh_mesher(verbose=True)
m.add_rectangle(0, 0, 0, 1e-3, 1e-3, mesh_size=0.2e-3, name="pad")
m.add_cylinder_surface(...)        # TSV / 过孔表面
m.add_via_with_pads(...)           # 带 pad 的过孔
verts, tris = m.get_arrays()
finalize_gmsh_mesher(m)
```

详见 [docs/UCIE_IMPLEMENTATION.md](docs/UCIE_IMPLEMENTATION.md) 与
[docs/VERTICAL_CURRENT_SUPPORT.md](docs/VERTICAL_CURRENT_SUPPORT.md)。

## 8. API 速查表

`import mom` 顶层导出：

| 符号 | 说明 |
|------|------|
| `Microstrip` | 微带线 MoM 求解器（`solve` / `solve_sweep` / `to_touchstone`） |
| `Structure` | 任意导体求解器（RWG 网格 + 并矢格林 + Schur N 端口） |
| `RectangleConductor` | `Structure` 用的矩形带导体描述符 |
| `Stackup` | `Structure` 用的单层介质 + PEC 接地描述符 |
| `FreqSweep` | 线性 / 对数扫频配置（`.linear` / `.logarithmic` 类方法） |
| `solve_microstrip` / `solve_microstrip_sweep` / `MicrostripConfig` | 低层微带求解 |
| `write_touchstone` / `read_touchstone` | Touchstone 2.0 `.sNp` 读写（RI/MA/DB） |
| `square_in_place` / `SweepScale` | 零拷贝 NumPy 通路 / 扫频刻度枚举 |
| `__version__` | 版本号 |

网格生成在 `mom.gmsh_mesh`（需显式 `import mom.gmsh_mesh`）：
`GmshMesher`、`UCieStructure`、`build_ucie_32bit_mesh`、
`create_gmsh_mesher`、`finalize_gmsh_mesher`。

## 9. 常见问题

**Q: 构建时报错 "Could not find Eigen"？**
A: Eigen 由 CMake `FetchContent` 在首次配置时从网络拉取。请确认联网；若在内网，
可设 `FETCHCONTENT_SOURCE_DIR_EIGEN` 指向本地 Eigen 源码目录。

**Q: pybind11 3.0 报静态初始化崩溃？**
A: `pyproject.toml` 已锁定 `pybind11>=2.13,<3`，部分 Anaconda Python（旧 CRT）
与 3.0.x 冲突。保持该约束即可。

**Q: 求解很慢？**
A: ① 增大 `nx`（微带）或细化网格会显著增加 RWG 基函数数，先用小网格验证再加密；
② 确认 OpenMP 已启用（CMake 配置日志会打印 `OpenMP: ON`）；③ 大网格用 pFFT 求解器。

**Q: 如何选端口？**
A: `add_port(name, basis_index)` 在指定 RWG 基函数（导体表面某条边）上定义端口。
对微带线，通常选首尾两端的基函数（`0` 与 `-1`）。多端口结构可加任意个端口。

**Q: 输出单位？**
A: `write_touchstone` 默认 `freq_unit="GHz"`、`fmt="RI"`（实虚部），参考阻抗 50 Ω。
支持 `MA`（幅角）、`DB`（dB-幅角）与 `Hz/kHz/MHz/GHz` 频率单位。

**Q: 支持哪些平台？**
A: 源码可在 Windows（MSVC/MinGW）、Linux、macOS 上编译。预编译二进制不随包提供，
请用 `pip install -e .` 在本机构建。
